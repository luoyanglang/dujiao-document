# 卡密管理

> 更新时间：2026-03-28

卡密（Card Secret）是 Dujiao-Next 自动交付类商品的核心。用户支付成功后，系统自动从库存中分配卡密并展示给用户。本指南介绍卡密的导入、管理和交付流程。

---

## 1. 基本概念

| 概念 | 说明 |
|------|------|
| 卡密 | 待交付的数字商品内容（如激活码、账号密码、CDKey 等） |
| 批次 | 一次导入操作产生的卡密集合，便于分组管理 |
| 自动交付 | 用户支付后系统自动分配卡密并展示 |
| 手动交付 | 管理员人工填写交付内容（不使用卡密库存） |

---

## 2. 卡密状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 可用 | `available` | 待分配，可被新订单使用 |
| 已预留 | `reserved` | 已关联订单，等待支付完成 |
| 已使用 | `used` | 已交付给用户 |

### 状态流转

```
导入 → 可用（available）
         ↓ 用户下单
       已预留（reserved）
         ↓ 支付成功 & 自动交付
       已使用（used）

支付超时/订单取消 → 释放回 可用（available）
```

---

## 3. 导入卡密

### 3.1 手动输入

后台「卡密管理」页面：

1. 选择商品（必须是自动交付类型）
2. 选择 SKU（如果商品有多个 SKU）
3. 在文本框中输入卡密，每行一条
4. 可选填写批次名称和备注
5. 点击「导入」

> 系统会自动去除空行、首尾空格，并去重。

### 3.2 CSV 导入

支持通过 CSV 文件批量导入：

**CSV 格式要求：**

- 必须包含 `secret` 列（表头不区分大小写）
- 每行一条卡密
- 系统自动跳过空行和重复项
- 支持 UTF-8 编码（自动处理 BOM 头）

**CSV 示例：**

```csv
secret
ABCD-1234-EFGH-5678
IJKL-9012-MNOP-3456
QRST-7890-UVWX-1234
```

### 3.3 批次信息

每次导入自动生成批次记录，包含：

- 批次号（自动生成，格式：`BATCH-{时间戳}-{随机串}`）
- 导入来源（手动 / CSV）
- 卡密总数
- 各状态数量统计

---

## 4. 管理卡密

### 4.1 查看卡密

支持多维度筛选：

| 筛选条件 | 说明 |
|---------|------|
| 商品 | 按所属商品筛选 |
| SKU | 按 SKU 变体筛选 |
| 批次 | 按导入批次筛选 |
| 状态 | 按可用/已预留/已使用筛选 |
| 关键词 | 按卡密内容模糊搜索 |
| 批次号 | 按批次号搜索 |

### 4.2 批量操作

- **批量修改状态**：选择多条卡密批量切换状态
- **批量删除**：删除选中的卡密（已使用的不建议删除）
- **按批次操作**：对整个批次执行状态变更

### 4.3 导出卡密

支持两种格式导出：

| 格式 | 说明 |
|------|------|
| TXT | 每行一个卡密内容，适合简单使用 |
| CSV | 包含完整字段（ID、卡密、状态、商品、SKU、订单、批次、时间等） |

---

## 5. 库存统计

### 5.1 商品维度

在商品列表和详情页可查看：

- **总库存**：所有卡密数量
- **可用库存**：状态为 `available` 的卡密数量
- **已预留**：关联待支付订单的卡密数量
- **已售出**：已交付给用户的卡密数量

### 5.2 批次维度

批次列表展示每个批次的统计信息：

- 批次号和名称
- 导入来源和时间
- 各状态数量
- 所属商品和 SKU

---

## 6. 自动交付流程

用户支付成功后，系统自动执行以下步骤：

1. 确认订单状态为「已支付」
2. 检查所有商品项的交付类型为 `auto`
3. 按商品和 SKU 分配卡密：
   - 优先使用已预留（`reserved`）的卡密
   - 不足时从可用（`available`）卡密中补充
4. 将分配的卡密标记为已使用（`used`）
5. 生成交付记录，卡密内容作为交付内容展示给用户
6. 订单状态更新为「已完成」

> 如果可用卡密不足，自动交付会失败，需要管理员补充卡密后手动处理。

---

## 7. 注意事项

- 自动交付商品必须先导入卡密才能正常售卖
- 卡密内容在导入时不做加密存储，请注意数据安全
- 批量导入时系统自动去重，相同内容的卡密不会重复导入
- 商品有多个 SKU 时，卡密按 SKU 维度独立管理
- 建议定期检查库存，及时补充即将售罄的商品卡密
- 库存不足时仪表盘会显示预警提示
