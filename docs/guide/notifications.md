# 通知中心配置

> 更新时间：2026-03-28

Dujiao-Next 通知中心支持通过邮件和 Telegram 两种渠道，在关键业务事件发生时自动通知管理员。

---

## 1. 功能概览

| 功能 | 说明 |
|------|------|
| 多渠道通知 | 支持邮件和 Telegram 两种通知渠道 |
| 事件驱动 | 订单支付、充值成功、待交付提醒、异常预警等 |
| 自定义模板 | 每种事件可自定义通知标题和内容 |
| 多语言 | 模板支持简体中文、繁体中文、英文 |
| 去重机制 | 短时间内相同事件不重复通知 |
| 库存预警 | 自动检测商品库存不足并发送告警 |

---

## 2. 配置入口

后台 → 设置 → 通知中心

---

## 3. 通知渠道

### 3.1 邮件通知

| 字段 | 说明 |
|------|------|
| 是否启用 | 邮件通知开关 |
| 收件人列表 | 接收通知的邮箱地址，支持多个 |

> 邮件通知依赖 SMTP 配置，请先在「设置 → SMTP 邮件」中完成邮件服务器配置。

### 3.2 Telegram 通知

| 字段 | 说明 |
|------|------|
| 是否启用 | Telegram 通知开关 |
| Chat ID 列表 | 接收通知的 Telegram Chat ID，支持多个 |

> 需要在系统中配置 Telegram Bot Token。可以通过「设置 → Telegram 登录」中的 Bot Token，或通过 Telegram Bot 服务的渠道客户端 Token。

**获取 Chat ID 的方法：**

1. 向你的 Bot 发送一条消息
2. 访问 `https://api.telegram.org/bot{你的BotToken}/getUpdates`
3. 在返回结果中找到 `chat.id` 字段

## 4. 通知事件

### 4.1 事件类型

| 事件 | 标识 | 说明 |
|------|------|------|
| 钱包充值成功 | `wallet_recharge_success` | 用户完成钱包充值后通知 |
| 订单支付成功 | `order_paid_success` | 用户完成订单支付后通知 |
| 待人工交付 | `manual_fulfillment_pending` | 人工交付类订单支付后提醒管理员处理 |
| 异常预警 | `exception_alert` | 库存不足、订单积压等异常情况 |

每个事件可独立启用/禁用。

### 4.2 异常预警详情

系统定期检查以下指标，超过阈值时自动触发预警：

| 预警类型 | 说明 |
|---------|------|
| 商品售罄 | 有商品的可用库存为 0 |
| 库存不足 | 商品库存低于预警线 |
| 待支付积压 | 待支付订单数量异常 |
| 支付失败 | 支付失败次数异常 |

预警通知会包含受影响的商品列表和具体数据。

## 5. 自定义模板

每种通知事件支持三个语言版本的模板自定义（简体中文 / 繁体中文 / 英文）。

### 5.1 模板字段

每个模板包含：

- **标题**：通知的标题/主题
- **内容**：通知的详细正文

### 5.2 模板变量

<div v-pre>

模板中可使用 `{{变量名}}` 插入动态数据：

**钱包充值成功：**

| 变量 | 说明 |
|------|------|
| `{{customer_label}}` | 用户名称和邮箱 |
| `{{customer_email}}` | 用户邮箱 |
| `{{recharge_no}}` | 充值单号 |
| `{{amount}}` | 充值金额 |
| `{{currency}}` | 货币 |
| `{{payment_channel}}` | 支付渠道 |

**订单支付成功：**

| 变量 | 说明 |
|------|------|
| `{{customer_label}}` | 用户名称和邮箱 |
| `{{customer_email}}` | 用户邮箱 |
| `{{order_no}}` | 订单号 |
| `{{amount}}` | 订单金额 |
| `{{currency}}` | 货币 |
| `{{payment_channel}}` | 支付渠道 |
| `{{items_summary}}` | 商品清单（含规格） |
| `{{fulfillment_items_summary}}` | 待人工交付商品 |
| `{{delivery_summary}}` | 交付方式摘要 |

**待人工交付：**

| 变量 | 说明 |
|------|------|
| `{{customer_label}}` | 用户名称和邮箱 |
| `{{customer_email}}` | 用户邮箱 |
| `{{order_no}}` | 订单号 |
| `{{order_status}}` | 订单当前状态 |
| `{{fulfillment_items_summary}}` | 待交付商品清单 |
| `{{delivery_summary}}` | 交付信息 |

**异常预警：**

| 变量 | 说明 |
|------|------|
| `{{alert_type}}` | 预警类型 |
| `{{alert_level}}` | 预警级别（error / warning） |
| `{{alert_value}}` | 当前值 |
| `{{alert_threshold}}` | 阈值 |
| `{{message}}` | 预警消息 |
| `{{affected_items_summary}}` | 受影响商品明细 |
| `{{affected_items_count}}` | 受影响项数 |

## 6. 高级设置

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 默认语言 | `zh-CN` | 通知模板使用的默认语言 |
| 去重间隔 | 300 秒 | 同一事件在此时间内不重复发送（30-86400 秒） |
| 库存预警间隔 | 1800 秒 | 库存预警的最小发送间隔（60-604800 秒） |
| 忽略商品 | 无 | 指定不参与库存预警检查的商品 ID 列表 |

---

## 7. 通知日志

后台可查看通知发送历史，包含：

- 事件类型
- 发送渠道（邮件 / Telegram）
- 发送状态（成功 / 失败）
- 发送时间

支持发送测试通知验证配置是否正确。
