# 后台管理入门

> 更新时间：2026-03-28

本指南介绍 Dujiao-Next 后台管理面板的核心功能与操作流程，帮助管理员快速上手日常运营。

---

## 1. 登录后台

- 默认地址：`http://localhost:5174`（开发环境）或你部署的管理后台域名
- 使用管理员账号密码登录
- 首次部署时管理员账号在 `config.yml` 的 `bootstrap` 节点配置

---

## 2. 仪表盘（Dashboard）

登录后进入仪表盘概览页面，提供以下关键数据：

### 2.1 核心指标

| 指标 | 说明 |
|------|------|
| 订单总数 | 期间内创建的订单数 |
| 已支付订单 | 成功完成支付的订单数 |
| 已完成订单 | 已交付/已完成的订单数 |
| 待支付订单 | 等待付款的订单数 |
| GMV（成交额）| 已支付订单的总金额 |
| 总成本 | 已售商品的成本总额 |
| 利润 | GMV - 总成本 |
| 利润率 | 利润 / GMV × 100% |
| 支付成功率 | 成功支付数 / 支付发起数 |
| 新增用户 | 期间内新注册的用户数 |

### 2.2 转化漏斗

仪表盘展示从下单到完成的全链路转化：

```
创建订单 → 发起支付 → 支付成功 → 订单已支付 → 订单完成
```

包含「支付转化率」和「完成率」两个关键转化指标。

### 2.3 库存预警

系统自动检测并提示：

- 已售罄的商品数量
- 库存不足的商品数量（含 SKU 维度）
- 待支付订单积压
- 支付失败率异常

### 2.4 排行与趋势

- **趋势图**：按日展示订单数、支付金额、利润等变化
- **商品排行**：按销量和金额排序的热销商品
- **支付渠道排行**：各支付渠道的使用频率和转化率

> 仪表盘数据有 45 秒缓存，可点击「刷新」获取最新数据。支持按时间范围筛选（今天 / 7 天 / 30 天 / 自定义，最长 90 天）。

## 3. 商品管理

### 3.1 创建商品

后台「商品管理」页面创建商品，主要字段：

| 字段 | 说明 |
|------|------|
| 分类 | 所属商品分类 |
| 标识（Slug） | URL 唯一标识，用于前台访问 |
| 标题 / 描述 / 详情 | 支持多语言（zh-CN / zh-TW / en-US） |
| 价格 | 商品基础售价 |
| 成本价 | 用于利润计算 |
| 图片 | 商品展示图片（支持多张） |
| 标签 | 用于搜索和筛选 |
| 购买类型 | `guest`（允许游客购买）或 `member`（仅注册用户） |
| 单次购买上限 | 每次下单最多购买数量（0 = 不限） |
| 交付类型 | `auto`（自动发卡）或 `manual`（人工交付） |
| 手动库存 | 交付类型为 `manual` 时的库存数量（-1 = 无限） |
| 允许分销 | 是否参与分销佣金计算 |
| 排序 | 数值越大排列越靠前 |

### 3.2 SKU 变体

商品支持多个 SKU 变体（如不同规格、时长等）：

- 每个 SKU 有独立的价格、成本价和库存
- SKU 可独立启用/禁用
- 自动交付商品的库存按 SKU 维度的卡密数量计算

### 3.3 交付类型说明

| 类型 | 说明 | 库存管理 |
|------|------|---------|
| `auto` | 支付后自动发送卡密 | 由卡密数量决定 |
| `manual` | 管理员手动交付 | 手动设置库存数 |

### 3.4 SEO 设置

每个商品可配置多语言 SEO 元数据，优化搜索引擎收录。

## 5. 订单管理

### 5.1 订单状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 待支付 | `pending_payment` | 用户已下单，等待付款 |
| 已支付 | `paid` | 支付成功，等待交付 |
| 处理中 | `processing` | 正在交付处理 |
| 已交付 | `delivered` | 人工交付完成 |
| 已完成 | `completed` | 自动发卡完成 |
| 已取消 | `canceled` | 用户取消或管理员取消 |
| 已过期 | `expired` | 超时未支付自动过期 |

### 5.2 订单操作

- **查看详情**：订单信息、商品明细、支付记录、交付记录
- **手动交付**：对 `manual` 类型商品手动填写交付内容
- **退款到钱包**：将已支付金额退回用户钱包（支持部分退款）
- **查看关联信息**：支付记录、佣金记录、对接订单等

### 5.3 多商品订单

包含多个商品的订单会自动拆分为父子订单：
- 父订单记录总体信息
- 子订单按商品/交付类型拆分
- 子订单独立交付，父订单状态随子订单同步更新

## 6. 支付管理

### 6.1 支付渠道配置

后台「支付渠道」页面配置各支付方式，详见 [支付配置指南](/payment/guide)。

### 6.2 支付记录

- 查看所有支付记录和状态
- 支持按时间、状态、渠道筛选
- 支持导出支付数据

## 7. 用户管理

### 7.1 用户列表

- 查看所有注册用户信息
- 按邮箱、状态搜索
- 查看用户登录日志

### 7.2 用户操作

| 操作 | 说明 |
|------|------|
| 启用/禁用 | 批量或单个切换用户状态 |
| 编辑资料 | 修改用户显示名称等信息 |
| 调整余额 | 手动增加或扣减钱包余额 |
| 设置等级 | 手动分配会员等级 |
| 查看钱包 | 查看用户余额和交易记录 |
| 查看优惠券 | 查看用户的优惠券使用情况 |

---

## 8. 内容管理

### 8.1 Banner 管理

- 创建首页轮播 Banner
- 支持多语言标题
- 可设置链接类型（无链接 / 站内链接 / 外部链接）
- 支持排序和启用/禁用

### 8.2 文章/公告

- 发布博客文章或站点公告
- 支持 Markdown 格式，多语言内容
- 文章类型：`blog`（博客）或 `notice`（公告）
- 支持发布/取消发布

---

## 9. 系统设置

后台「设置」页面集中管理全站配置：

| 设置项 | 说明 |
|--------|------|
| 站点配置 | 站点名称、货币、语言、功能开关等 |
| SMTP 邮件 | 邮件服务器配置，支持发送测试邮件 |
| 验证码 | 图片验证码或 Cloudflare Turnstile |
| Telegram 登录 | Telegram OAuth 登录配置 |
| 通知中心 | 邮件/Telegram 通知渠道配置 |
| 订单邮件模板 | 自定义订单状态通知邮件模板 |
| 分销设置 | 佣金比例、确认期、提现配置 |
| Telegram Bot | Bot 功能配置（需购买 Bot 服务） |

---

## 10. 权限管理（RBAC）

系统支持基于角色的权限控制：

### 10.1 管理员管理

- 创建、编辑、删除管理员账号
- 为管理员分配角色

### 10.2 角色与权限

- 创建自定义角色
- 为角色分配具体权限（如「仅查看订单」「管理商品」等）
- 查看系统可用权限目录

### 10.3 审计日志

所有管理员操作均有审计记录，包含：
- 操作人、操作类型
- 操作对象和变更内容
- IP 地址和时间戳
