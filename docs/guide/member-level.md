# 会员等级

> 更新时间：2026-03-28

Dujiao-Next 支持会员等级系统，为不同消费层级的用户提供专属定价和折扣。会员等级可自动升级（基于充值/消费累计），也可由管理员手动分配。

---

## 1. 功能概览

| 功能 | 说明 |
|------|------|
| 等级管理 | 创建多个等级，设置折扣率和升级门槛 |
| 等级专属价 | 为指定商品/SKU 设置会员专享价格 |
| 自动升级 | 充值或消费累计达标后自动升级 |
| 手动分配 | 管理员可直接设置用户等级 |
| 等级折扣 | 全局折扣率自动应用于所有商品 |

---

## 2. 等级配置

### 2.1 创建等级

后台「会员等级」页面创建，每个等级包含：

| 字段 | 说明 |
|------|------|
| 名称 | 等级名称（支持多语言） |
| 标识 (Slug) | 唯一标识，如 `default`、`silver`、`gold`、`diamond` |
| 图标 | Emoji 或图标 URL |
| 折扣率 | 0-100 的数值。100 = 无折扣，90 = 九折（10% off），80 = 八折（20% off） |
| 充值门槛 | 累计充值达到此金额可自动升级（0 = 不以充值为条件） |
| 消费门槛 | 累计消费达到此金额可自动升级（0 = 不以消费为条件） |
| 排序权重 | 数值越大等级越高 |
| 默认等级 | 新用户自动分配的等级（仅一个） |
| 是否启用 | 开关 |

### 2.2 等级示例

```
默认会员（default）  折扣率 100  排序 0   → 无折扣
白银会员（silver）   折扣率 95   排序 10  → 充值满 500 或消费满 1000
黄金会员（gold）     折扣率 90   排序 20  → 充值满 2000 或消费满 5000
钻石会员（diamond）  折扣率 85   排序 30  → 充值满 5000 或消费满 10000
```

---

## 3. 价格计算

会员价格按以下优先级确定：

### 3.1 优先级

1. **SKU 级专属价**：为特定 SKU 设置的会员价（最高优先）
2. **商品级专属价**：为商品设置的会员价
3. **全局折扣率**：按等级折扣率计算 → `原价 × 折扣率 / 100`

> 只有会员价低于原价时才会生效，不会出现会员价高于原价的情况。

### 3.2 设置专属价

后台「会员等级价格」页面：

- 选择会员等级和商品
- 可为商品设置统一会员价
- 也可为商品下的具体 SKU 单独设置会员价
- SKU 级别的设置优先于商品级别

---

## 4. 自动升级

### 4.1 升级条件

用户达到以下**任一**条件即可升级：

- 累计充值金额 ≥ 等级的「充值门槛」（门槛 > 0 时生效）
- 累计消费金额 ≥ 等级的「消费门槛」（门槛 > 0 时生效）

### 4.2 升级时机

以下操作完成后系统自动检查并升级：

- 钱包充值成功
- 订单支付成功

### 4.3 升级规则

- **只升不降**：等级只会向上升级，不会因后续消费减少而降级
- **跨级升级**：如果累计金额直接达到高等级门槛，会跳过中间等级
- 按排序权重从高到低匹配，取满足条件的最高等级

---

## 5. 管理员操作

### 5.1 手动设置等级

后台「用户管理」中可直接设置用户的会员等级，不受门槛限制。

### 5.2 批量回填

对于系统升级前已存在的老用户，可使用「批量回填默认等级」功能，为尚未分配等级的用户统一设置默认等级。

---

## 6. 注意事项

- 每个系统只能有一个默认等级
- 默认等级不可删除
- 等级标识（Slug）创建后不可重复
- 折扣率设为 100 或以上表示该等级无折扣
- 门槛设为 0 表示该条件不参与判断
- 会员折扣在活动价之前应用，两者可叠加
