# 优惠券与活动价

> 更新时间：2026-03-28

Dujiao-Next 提供两种促销机制：**优惠券**（用户主动输入码使用）和**活动价**（满足条件自动应用）。两者可与会员等级折扣叠加使用。

---

## 1. 折扣应用顺序

用户下单时，系统按以下顺序计算最终价格：

```
基础价格（或 SKU 价格）
    ↓
会员等级折扣（自动应用）
    ↓
活动价（满足条件自动应用）
    ↓
优惠券折扣（用户主动使用）
    ↓
最终支付金额
```

> 活动价与会员价取最低价；优惠券在活动价基础上再扣减。

---

## 2. 优惠券

### 2.1 优惠券类型

| 类型 | 标识 | 计算方式 |
|------|------|---------|
| 固定金额减免 | `fixed` | 订单减免固定金额 |
| 百分比折扣 | `percent` | 订单减免百分比金额 |

### 2.2 创建优惠券

后台「优惠券」页面创建，配置项包括：

| 字段 | 说明 |
|------|------|
| 优惠码 | 唯一标识，用户输入此码使用 |
| 优惠类型 | `fixed`（固定金额）或 `percent`（百分比） |
| 优惠值 | 固定金额数值或折扣百分比 |
| 最低消费 | 订单中符合条件商品的小计需达到此金额 |
| 最大折扣 | 百分比类型的折扣上限金额 |
| 总使用次数 | 全站累计可用次数，`0` 为不限 |
| 每人限用 | 每个用户可使用次数，`0` 为不限 |
| 适用商品 | 指定适用的商品范围（可选全部或指定商品） |
| 有效期 | 开始时间与结束时间 |
| 是否启用 | 开关 |

### 2.3 使用规则

用户下单时输入优惠码，系统按以下顺序校验：

1. 优惠码是否存在且已启用
2. 是否在有效期内（开始时间 ≤ 当前 ≤ 结束时间）
3. 全站总使用次数是否未超限
4. 当前用户使用次数是否未超限（注册用户）
5. 符合条件商品的小计是否达到最低消费

**折扣计算：**

- **固定金额**：直接减免设定的金额
- **百分比**：计算符合条件商品小计的百分比，受「最大折扣」上限约束
- 折扣金额不会超过符合条件商品的小计

### 2.4 适用范围

优惠券的「适用商品」决定了哪些商品参与折扣计算：

- 只有在范围内的商品小计会用于判断「最低消费」
- 只有范围内的商品金额会参与折扣计算
- 订单中不在范围内的商品不受影响

## 3. 活动价

### 3.1 活动价类型

| 类型 | 标识 | 计算方式 |
|------|------|---------|
| 百分比折扣 | `percent` | 单价 = 原价 × (100 - 值) / 100 |
| 固定金额减免 | `fixed` | 单价 = 原价 - 值（最低为 0） |
| 直降特价 | `special_price` | 单价 = 设定值 |

### 3.2 创建活动价

后台「活动价」页面创建，配置项包括：

| 字段 | 说明 |
|------|------|
| 名称 | 活动名称（支持多语言） |
| 适用商品 | 指定适用的商品 |
| 优惠类型 | `percent` / `fixed` / `special_price` |
| 优惠值 | 对应类型的数值 |
| 门槛金额 | 购买小计（单价 × 数量）需达到此金额才生效 |
| 有效期 | 开始时间与结束时间 |
| 是否启用 | 开关 |

### 3.3 阶梯折扣

同一商品可配置多条活动规则，形成阶梯折扣：

```
规则 A：门槛 50 元，优惠 1%
规则 B：门槛 150 元，优惠 2%
规则 C：门槛 500 元，优惠 5%
```

系统从高到低匹配门槛，取满足条件的最高档规则：

- 购买小计 49 元 → 无折扣
- 购买小计 100 元 → 匹配规则 A（1%）
- 购买小计 200 元 → 匹配规则 B（2%）
- 购买小计 600 元 → 匹配规则 C（5%）

### 3.4 应用逻辑

- 活动价按**商品维度**匹配，每件商品独立计算
- 系统在前台商品详情页展示活动规则，即使当前数量未满足门槛也会显示提示
- 价格精度为两位小数
- 活动价不与同类型活动价叠加，同一商品取最优匹配

---

## 4. 管理建议

- 优惠券适合限时促销、新用户引流、指定商品推广等场景
- 活动价适合阶梯定价、批量采购优惠等场景
- 两者可同时使用：活动价自动降低单价，优惠券在此基础上再减
- 建议定期清理过期的优惠券和活动价，保持列表清晰
