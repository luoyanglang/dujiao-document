# 更新日志

## [v0.0.7-beta] - 2026-02-28

### 更新内容：
- 新增推广返利系统：用户可开通联盟ID进行推广，支持点击归因、佣金累计与提现流程。
- 支付渠道能力扩展：新增 TokenPay，并优化 Bepusdt 配置方式，支付接入更灵活。
- 优化库存扣减与售罄状态：库存与下单状态联动更准确，减少前后台显示不一致问题。
- 优化支付页面与支付流程体验：下单支付步骤更清晰，操作反馈更直观。
- 优化初始管理员配置流程：首次部署时管理员初始化配置更稳定、兼容性更好。

## [v0.0.6-beta] - 2026-02-27

### 更新内容：
- 优化钱包充值状态体验：长时间未支付的充值单会按规则过期，状态展示更清晰。
- 提升充值回调稳定性：支付结果同步更稳，减少重复回调导致的状态异常。
- 优化订单状态流转：后台订单状态更新体验更顺畅，状态一致性更好。
- 持续优化多语言与系统稳定性：细节文案与流程体验进一步统一。

## [v0.0.5-beta] - 2026-02-25

### 更新内容：
- 新增礼品卡体系：后台支持礼品卡生成、状态管理、筛选、批量状态更新与导出；前台用户中心新增礼品卡兑换页（含动效），支持按验证码场景开关校验（图片验证码/Turnstile），兑换后自动充入钱包余额。
- 礼品卡币种策略调整：后台生成礼品卡时币种自动继承全站币种，移除管理员手动输入币种，避免币种配置不一致。
- 支付回调安全热修：回调入口仅接受已识别并通过验签的渠道回调（微信/支付宝/易支付/EPUSDT），阻断伪造回调将订单或充值单错误标记为已支付；同时保留 `GET /payments/callback` 兼容易支付回调，未识别请求统一返回 `fail` 且不会更新支付状态。

## [v0.0.4-beta] - 2026-02-25

### 更新内容：
- 新增商品分类图标能力：后台支持上传分类图标，前台分类侧栏与商品卡片支持展示分类图标。
- 新增 SKU 全链路支持：后台商品管理、库存展示与前台下单流程统一到 SKU 维度。
- 优化自动交付库存逻辑：自动发货商品库存统计与展示规则保持一致，并修复自动库存统计入参问题。
- 增强折扣与结算防呆：强化活动配置提醒与订单金额校验，降低误配置导致异常价格或 0 元单的风险。
- 增强管理员退款校验：后台退款到余额前增加订单支付状态校验，避免未支付订单被误退款。
- 修复支付跳转异常：修复部分场景支付完成后的跳转问题。
- 修复前端旧资源请求问题：减少发布后请求旧 chunk 导致的页面报错。
- 统一全站币种字段：前后台与接口币种字段与展示逻辑统一。
- 新增 SEO Meta 配置能力：支持配置更完整的站点 SEO 元信息。
- 体验与稳定性优化：包含后台管理、卡密管理、购物车体验与邮件发送链路等多项改进。

## [v0.0.3-beta] - 2026-02-22

### 更新内容：
- 新增用户钱包体系：支持余额、充值、钱包明细流水记录（含充值、支付、退款回退等资金动向）。
- 升级订单支付能力：支持仅余额支付、余额+在线支付组合支付、仅在线支付；订单未支付成功或取消时自动退回已扣余额。
- 后台新增钱包管理能力：支持查看用户余额与明细、手动增减用户余额、订单退款仅退回用户余额。
- 新增 Telegram 登录能力：首次 Telegram 登录且未绑定账号时自动创建账号并直接登录。
- 安全中心新增 Telegram 绑定/解绑：已登录用户可绑定或解绑 Telegram；未绑定真实邮箱的 Telegram 新账号不允许解绑。
- 完善账户安全策略：新增 `email_change_mode` 与 `password_change_mode` 返回字段，支持“仅绑定新邮箱”与“首次设置密码无需旧密码”流程。
- 优化用户中心导航：左侧菜单顺序调整为“概览 / 我的订单 / 我的钱包 / 安全中心 / 资料设置”，并补充菜单图标。
