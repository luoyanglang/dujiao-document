# 後台管理入門

> 更新時間：2026-03-28

本指南介紹 Dujiao-Next 後台管理面板的核心功能與操作流程，幫助管理員快速上手日常營運。

---

## 1. 登入後台

- 預設地址：`http://localhost:5174`（開發環境）或你部署的管理後台網域
- 使用管理員帳號密碼登入
- 首次部署時管理員帳號在 `config.yml` 的 `bootstrap` 節點設定

---

## 2. 儀表盤（Dashboard）

登入後進入儀表盤概覽頁面，提供以下關鍵數據：

### 2.1 核心指標

| 指標 | 說明 |
|------|------|
| 訂單總數 | 期間內建立的訂單數 |
| 已支付訂單 | 成功完成支付的訂單數 |
| 已完成訂單 | 已交付/已完成的訂單數 |
| 待支付訂單 | 等待付款的訂單數 |
| GMV（成交額）| 已支付訂單的總金額 |
| 總成本 | 已售商品的成本總額 |
| 利潤 | GMV - 總成本 |
| 利潤率 | 利潤 / GMV × 100% |
| 支付成功率 | 成功支付數 / 支付發起數 |
| 新增使用者 | 期間內新註冊的使用者數 |

### 2.2 轉化漏斗

儀表盤展示從下單到完成的全鏈路轉化：

```
建立訂單 → 發起支付 → 支付成功 → 訂單已支付 → 訂單完成
```

包含「支付轉化率」和「完成率」兩個關鍵轉化指標。

### 2.3 庫存預警

系統自動檢測並提示：

- 已售罄的商品數量
- 庫存不足的商品數量（含 SKU 維度）
- 待支付訂單積壓
- 支付失敗率異常

### 2.4 排行與趨勢

- **趨勢圖**：按日展示訂單數、支付金額、利潤等變化
- **商品排行**：按銷量和金額排序的熱銷商品
- **支付渠道排行**：各支付渠道的使用頻率和轉化率

> 儀表盤數據有 45 秒快取，可點擊「重新整理」取得最新數據。支持按時間範圍篩選（今天 / 7 天 / 30 天 / 自訂，最長 90 天）。

## 3. 商品管理

### 3.1 建立商品

後台「商品管理」頁面建立商品，主要欄位：

| 欄位 | 說明 |
|------|------|
| 分類 | 所屬商品分類 |
| 標識（Slug） | URL 唯一標識，用於前台存取 |
| 標題 / 描述 / 詳情 | 支持多語言（zh-CN / zh-TW / en-US） |
| 價格 | 商品基礎售價 |
| 成本價 | 用於利潤計算 |
| 圖片 | 商品展示圖片（支持多張） |
| 標籤 | 用於搜尋和篩選 |
| 購買類型 | `guest`（允許訪客購買）或 `member`（僅註冊使用者） |
| 單次購買上限 | 每次下單最多購買數量（0 = 不限） |
| 交付類型 | `auto`（自動發卡）或 `manual`（人工交付） |
| 手動庫存 | 交付類型為 `manual` 時的庫存數量（-1 = 無限） |
| 允許分銷 | 是否參與分銷佣金計算 |
| 排序 | 數值越大排列越靠前 |

### 3.2 SKU 變體

商品支持多個 SKU 變體（如不同規格、時長等）：

- 每個 SKU 有獨立的價格、成本價和庫存
- SKU 可獨立啟用/停用
- 自動交付商品的庫存按 SKU 維度的卡密數量計算

### 3.3 交付類型說明

| 類型 | 說明 | 庫存管理 |
|------|------|---------|
| `auto` | 支付後自動發送卡密 | 由卡密數量決定 |
| `manual` | 管理員手動交付 | 手動設定庫存數 |

### 3.4 SEO 設定

每個商品可設定多語言 SEO 中繼資料，優化搜尋引擎收錄。

## 4. 分類管理

- 建立和管理商品分類
- 分類名稱支持多語言
- 可設定分類圖示和排序
- 商品必須歸屬於一個分類

---

## 5. 訂單管理

### 5.1 訂單狀態

| 狀態 | 標識 | 說明 |
|------|------|------|
| 待支付 | `pending_payment` | 使用者已下單，等待付款 |
| 已支付 | `paid` | 支付成功，等待交付 |
| 處理中 | `processing` | 正在交付處理 |
| 已交付 | `delivered` | 人工交付完成 |
| 已完成 | `completed` | 自動發卡完成 |
| 已取消 | `canceled` | 使用者取消或管理員取消 |
| 已過期 | `expired` | 逾時未支付自動過期 |

### 5.2 訂單操作

- **查看詳情**：訂單資訊、商品明細、支付記錄、交付記錄
- **手動交付**：對 `manual` 類型商品手動填寫交付內容
- **退款到錢包**：將已支付金額退回使用者錢包（支持部分退款）
- **查看關聯資訊**：支付記錄、佣金記錄、對接訂單等

### 5.3 多商品訂單

包含多個商品的訂單會自動拆分為父子訂單：
- 父訂單記錄整體資訊
- 子訂單按商品/交付類型拆分
- 子訂單獨立交付，父訂單狀態隨子訂單同步更新

---

## 6. 支付管理

### 6.1 支付渠道設定

後台「支付渠道」頁面設定各支付方式，詳見[支付設定指南](/zh-hant/payment/guide)。

### 6.2 支付記錄

- 查看所有支付記錄和狀態
- 支持按時間、狀態、渠道篩選
- 支持匯出支付資料

---

## 7. 使用者管理

### 7.1 使用者列表

- 查看所有註冊使用者資訊
- 按電子郵件、狀態搜尋
- 查看使用者登入日誌

### 7.2 使用者操作

| 操作 | 說明 |
|------|------|
| 啟用/停用 | 批量或單個切換使用者狀態 |
| 編輯資料 | 修改使用者顯示名稱等資訊 |
| 調整餘額 | 手動增加或扣減錢包餘額 |
| 設定等級 | 手動分配會員等級 |
| 查看錢包 | 查看使用者餘額和交易記錄 |
| 查看優惠券 | 查看使用者的優惠券使用情況 |

---

## 8. 內容管理

### 8.1 Banner 管理

- 建立首頁輪播 Banner
- 支持多語言標題
- 可設定連結類型（無連結 / 站內連結 / 外部連結）
- 支持排序和啟用/停用

### 8.2 文章/公告

- 發佈部落格文章或站點公告
- 支持 Markdown 格式，多語言內容
- 文章類型：`blog`（部落格）或 `notice`（公告）
- 支持發佈/取消發佈

---

## 9. 系統設定

後台「設定」頁面集中管理全站設定：

| 設定項 | 說明 |
|--------|------|
| 站點設定 | 站點名稱、貨幣、語言、功能開關等 |
| SMTP 郵件 | 郵件伺服器設定，支持傳送測試郵件 |
| 驗證碼 | 圖片驗證碼或 Cloudflare Turnstile |
| Telegram 登入 | Telegram OAuth 登入設定 |
| 通知中心 | 郵件/Telegram 通知渠道設定 |
| 訂單郵件範本 | 自訂訂單狀態通知郵件範本 |
| 分銷設定 | 佣金比例、確認期、提現設定 |
| Telegram Bot | Bot 功能設定（需購買 Bot 服務） |

---

## 10. 權限管理（RBAC）

系統支持基於角色的權限控制：

### 10.1 管理員管理

- 建立、編輯、刪除管理員帳號
- 為管理員分配角色

### 10.2 角色與權限

- 建立自訂角色
- 為角色分配具體權限（如「僅查看訂單」「管理商品」等）
- 查看系統可用權限目錄

### 10.3 稽核日誌

所有管理員操作均有稽核記錄，包含：
- 操作人、操作類型
- 操作對象和變更內容
- IP 地址和時間戳
