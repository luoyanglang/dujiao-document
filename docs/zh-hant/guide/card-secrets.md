# 卡密管理

> 更新時間：2026-03-28

卡密（Card Secret）是 Dujiao-Next 自動交付類商品的核心。使用者支付成功後，系統自動從庫存中分配卡密並展示給使用者。本指南介紹卡密的匯入、管理和交付流程。

---

## 1. 基本概念

| 概念 | 說明 |
|------|------|
| 卡密 | 待交付的數位商品內容（如啟用碼、帳號密碼、CDKey 等） |
| 批次 | 一次匯入操作產生的卡密集合，便於分組管理 |
| 自動交付 | 使用者支付後系統自動分配卡密並展示 |
| 手動交付 | 管理員人工填寫交付內容（不使用卡密庫存） |

---

## 2. 卡密狀態

| 狀態 | 標識 | 說明 |
|------|------|------|
| 可用 | `available` | 待分配，可被新訂單使用 |
| 已預留 | `reserved` | 已關聯訂單，等待支付完成 |
| 已使用 | `used` | 已交付給使用者 |

### 狀態流轉

```
匯入 → 可用（available）
         ↓ 使用者下單
       已預留（reserved）
         ↓ 支付成功 & 自動交付
       已使用（used）

支付逾時/訂單取消 → 釋放回 可用（available）
```

---

## 3. 匯入卡密

### 3.1 手動輸入

後台「卡密管理」頁面：

1. 選擇商品（必須是自動交付類型）
2. 選擇 SKU（如果商品有多個 SKU）
3. 在文字方塊中輸入卡密，每行一條
4. 可選填寫批次名稱和備註
5. 點擊「匯入」

> 系統會自動去除空行、首尾空格，並去重。

### 3.2 CSV 匯入

支持透過 CSV 檔案批量匯入：

**CSV 格式要求：**

- 必須包含 `secret` 欄（表頭不區分大小寫）
- 每行一條卡密
- 系統自動跳過空行和重複項
- 支持 UTF-8 編碼（自動處理 BOM 頭）

**CSV 範例：**

```csv
secret
ABCD-1234-EFGH-5678
IJKL-9012-MNOP-3456
QRST-7890-UVWX-1234
```

### 3.3 批次資訊

每次匯入自動產生批次記錄，包含：

- 批次號（自動產生，格式：`BATCH-{時間戳}-{隨機串}`）
- 匯入來源（手動 / CSV）
- 卡密總數
- 各狀態數量統計

---

## 4. 管理卡密

### 4.1 查看卡密

支持多維度篩選：

| 篩選條件 | 說明 |
|---------|------|
| 商品 | 按所屬商品篩選 |
| SKU | 按 SKU 變體篩選 |
| 批次 | 按匯入批次篩選 |
| 狀態 | 按可用/已預留/已使用篩選 |
| 關鍵字 | 按卡密內容模糊搜尋 |
| 批次號 | 按批次號搜尋 |

### 4.2 批量操作

- **批量修改狀態**：選擇多條卡密批量切換狀態
- **批量刪除**：刪除選中的卡密（已使用的不建議刪除）
- **按批次操作**：對整個批次執行狀態變更

### 4.3 匯出卡密

支持兩種格式匯出：

| 格式 | 說明 |
|------|------|
| TXT | 每行一個卡密內容，適合簡單使用 |
| CSV | 包含完整欄位（ID、卡密、狀態、商品、SKU、訂單、批次、時間等） |

---

## 5. 庫存統計

### 5.1 商品維度

在商品列表和詳情頁可查看：

- **總庫存**：所有卡密數量
- **可用庫存**：狀態為 `available` 的卡密數量
- **已預留**：關聯待支付訂單的卡密數量
- **已售出**：已交付給使用者的卡密數量

### 5.2 批次維度

批次列表展示每個批次的統計資訊：

- 批次號和名稱
- 匯入來源和時間
- 各狀態數量
- 所屬商品和 SKU

---

## 6. 自動交付流程

使用者支付成功後，系統自動執行以下步驟：

1. 確認訂單狀態為「已支付」
2. 檢查所有商品項的交付類型為 `auto`
3. 按商品和 SKU 分配卡密：
   - 優先使用已預留（`reserved`）的卡密
   - 不足時從可用（`available`）卡密中補充
4. 將分配的卡密標記為已使用（`used`）
5. 產生交付記錄，卡密內容作為交付內容展示給使用者
6. 訂單狀態更新為「已完成」

> 如果可用卡密不足，自動交付會失敗，需要管理員補充卡密後手動處理。

---

## 7. 注意事項

- 自動交付商品必須先匯入卡密才能正常售賣
- 卡密內容在匯入時不做加密儲存，請注意資料安全
- 批量匯入時系統自動去重，相同內容的卡密不會重複匯入
- 商品有多個 SKU 時，卡密按 SKU 維度獨立管理
- 建議定期檢查庫存，及時補充即將售罄的商品卡密
- 庫存不足時儀表盤會顯示預警提示
