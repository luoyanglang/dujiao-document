# 優惠券與活動價

> 更新時間：2026-03-28

Dujiao-Next 提供兩種促銷機制：**優惠券**（使用者主動輸入碼使用）和**活動價**（滿足條件自動套用）。兩者可與會員等級折扣疊加使用。

---

## 1. 折扣套用順序

使用者下單時，系統按以下順序計算最終價格：

```
基礎價格（或 SKU 價格）
    ↓
會員等級折扣（自動套用）
    ↓
活動價（滿足條件自動套用）
    ↓
優惠券折扣（使用者主動使用）
    ↓
最終付款金額
```

> 活動價與會員價取最低價；優惠券在活動價基礎上再扣減。

---

## 2. 優惠券

### 2.1 優惠券類型

| 類型 | 標識 | 計算方式 |
|------|------|---------|
| 固定金額減免 | `fixed` | 訂單減免固定金額 |
| 百分比折扣 | `percent` | 訂單減免百分比金額 |

### 2.2 建立優惠券

後台「優惠券」頁面建立，設定項包括：

| 欄位 | 說明 |
|------|------|
| 優惠碼 | 唯一標識，使用者輸入此碼使用 |
| 優惠類型 | `fixed`（固定金額）或 `percent`（百分比） |
| 優惠值 | 固定金額數值或折扣百分比 |
| 最低消費 | 訂單中符合條件商品的小計需達到此金額 |
| 最大折扣 | 百分比類型的折扣上限金額 |
| 總使用次數 | 全站累計可用次數，`0` 為不限 |
| 每人限用 | 每個使用者可使用次數，`0` 為不限 |
| 適用商品 | 指定適用的商品範圍（可選全部或指定商品） |
| 有效期 | 開始時間與結束時間 |
| 是否啟用 | 開關 |

### 2.3 使用規則

使用者下單時輸入優惠碼，系統按以下順序驗證：

1. 優惠碼是否存在且已啟用
2. 是否在有效期內（開始時間 ≤ 目前 ≤ 結束時間）
3. 全站總使用次數是否未超限
4. 目前使用者使用次數是否未超限（註冊使用者）
5. 符合條件商品的小計是否達到最低消費

**折扣計算：**

- **固定金額**：直接減免設定的金額
- **百分比**：計算符合條件商品小計的百分比，受「最大折扣」上限約束
- 折扣金額不會超過符合條件商品的小計

### 2.4 適用範圍

優惠券的「適用商品」決定了哪些商品參與折扣計算：

- 只有在範圍內的商品小計會用於判斷「最低消費」
- 只有範圍內的商品金額會參與折扣計算
- 訂單中不在範圍內的商品不受影響

## 3. 活動價

### 3.1 活動價類型

| 類型 | 標識 | 計算方式 |
|------|------|---------|
| 百分比折扣 | `percent` | 單價 = 原價 × (100 - 值) / 100 |
| 固定金額減免 | `fixed` | 單價 = 原價 - 值（最低為 0） |
| 直降特價 | `special_price` | 單價 = 設定值 |

### 3.2 建立活動價

後台「活動價」頁面建立，設定項包括：

| 欄位 | 說明 |
|------|------|
| 名稱 | 活動名稱（支援多語言） |
| 適用商品 | 指定適用的商品 |
| 優惠類型 | `percent` / `fixed` / `special_price` |
| 優惠值 | 對應類型的數值 |
| 門檻金額 | 購買小計（單價 × 數量）需達到此金額才生效 |
| 有效期 | 開始時間與結束時間 |
| 是否啟用 | 開關 |

### 3.3 階梯折扣

同一商品可設定多條活動規則，形成階梯折扣：

```
規則 A：門檻 50 元，優惠 1%
規則 B：門檻 150 元，優惠 2%
規則 C：門檻 500 元，優惠 5%
```

系統從高到低比對門檻，取滿足條件的最高檔規則：

- 購買小計 49 元 → 無折扣
- 購買小計 100 元 → 比對規則 A（1%）
- 購買小計 200 元 → 比對規則 B（2%）
- 購買小計 600 元 → 比對規則 C（5%）

### 3.4 套用邏輯

- 活動價按**商品維度**比對，每件商品獨立計算
- 系統在前台商品詳情頁展示活動規則，即使目前數量未滿足門檻也會顯示提示
- 價格精度為兩位小數
- 活動價不與同類型活動價疊加，同一商品取最優比對

---

## 4. 管理建議

- 優惠券適合限時促銷、新使用者引流、指定商品推廣等情境
- 活動價適合階梯定價、批量採購優惠等情境
- 兩者可同時使用：活動價自動降低單價，優惠券在此基礎上再減
- 建議定期清理過期的優惠券和活動價，保持列表清晰
