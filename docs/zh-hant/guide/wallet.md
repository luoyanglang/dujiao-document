# 錢包系統

> 更新時間：2026-03-28

Dujiao-Next 內建錢包系統，使用者可以透過多種支付方式充值錢包餘額，下單時使用餘額付款，也可以透過禮品卡兌換增加餘額。管理員可手動調整使用者餘額或將訂單退款至錢包。

---

## 1. 功能概覽

| 功能 | 說明 |
|------|------|
| 餘額充值 | 使用者透過已設定的支付管道充值錢包 |
| 餘額付款 | 下單時可使用錢包餘額抵扣全部或部分金額 |
| 交易紀錄 | 每筆充值、消費、退款均有完整紀錄 |
| 禮品卡兌換 | 使用者輸入禮品卡碼兌換餘額 |
| 管理員調整 | 管理員可增加或扣減使用者餘額 |
| 退款到錢包 | 管理員可將已付款訂單的金額退回使用者錢包 |

---

## 2. 使用者操作

### 2.1 充值錢包

1. 在前台進入「錢包」頁面
2. 輸入充值金額（必須大於 0，最多兩位小數）
3. 選擇支付管道完成付款
4. 付款成功後餘額自動入帳

> 充值未付款將在逾時後自動過期（預設 15 分鐘，與訂單付款逾時一致）。

### 2.2 使用餘額付款

下單時系統會自動偵測錢包餘額：

- **餘額 >= 訂單金額**：可全額使用餘額付款，無需線上支付
- **餘額 < 訂單金額**：餘額抵扣部分金額，剩餘透過線上支付管道完成
- 訪客使用者不支援錢包付款

### 2.3 兌換禮品卡

1. 在前台進入「錢包」頁面
2. 輸入禮品卡碼
3. 兌換成功後面額自動充入錢包

> 禮品卡碼不區分大小寫。已使用、已停用或已過期的禮品卡無法兌換。

### 2.4 查看交易紀錄

錢包頁面可查看所有歷史交易，每筆紀錄包含：

- 交易類型（充值 / 訂單付款 / 退款 / 管理員調整 / 禮品卡兌換）
- 金額與方向（收入 / 支出）
- 交易前後餘額
- 關聯訂單號（如有）
- 時間與備註

---

## 3. 交易類型

| 類型標識 | 說明 | 方向 |
|---------|------|------|
| `recharge` | 使用者充值 | 收入 |
| `order_pay` | 訂單餘額付款 | 支出 |
| `admin_refund` | 管理員退款到錢包 | 收入 |
| `admin_adjust` | 管理員手動調整 | 收入/支出 |
| `gift_card` | 禮品卡兌換 | 收入 |

---

## 4. 管理員操作

### 4.1 調整使用者餘額

後台「使用者管理」中選擇使用者，可手動增加或扣減餘額：

- 正數為增加餘額，負數為扣減
- 金額不可為 0
- 操作後餘額不可為負

### 4.2 退款到錢包

後台「訂單管理」中，對已付款的訂單可執行「退款到錢包」：

- 輸入退款金額（不可超過訂單可退金額）
- 退款金額自動充入使用者錢包
- 支援多次部分退款，累計不超過訂單實付金額
- 訪客訂單不支援退款到錢包
- 若訂單關聯了分銷佣金，退款時會同步處理佣金扣減

---

## 5. 禮品卡管理

### 5.1 產生禮品卡

後台「禮品卡」頁面可批次產生：

- **名稱**：批次名稱，便於管理
- **面額**：每張禮品卡的金額（大於 0，最多兩位小數）
- **數量**：單次最多產生 10,000 張
- **到期時間**：可選，不設則永不過期

產生後每張禮品卡獲得唯一卡碼（格式：`GC` + 時間戳 + 序號 + 隨機串）。

### 5.2 禮品卡狀態

| 狀態 | 說明 |
|------|------|
| `active` | 可用，等待兌換 |
| `redeemed` | 已被使用者兌換 |
| `disabled` | 已停用，不可兌換 |

- 管理員可批次啟用/停用禮品卡
- 已兌換的禮品卡不可修改狀態或刪除

### 5.3 匯出禮品卡

支援匯出為：

- **TXT 格式**：每行一個卡碼
- **CSV 格式**：包含完整欄位（ID、批次號、名稱、卡碼、面額、狀態、兌換資訊等）

---

## 6. 注意事項

- 錢包帳戶在使用者首次存取時自動建立，無需手動開通
- 所有金額操作使用冪等參考（Reference），確保不會重複扣款或充值
- 餘額精度為兩位小數
- 預設貨幣跟隨系統設定（如 CNY、USD 等）
