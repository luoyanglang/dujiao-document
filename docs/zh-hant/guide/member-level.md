# 會員等級

> 更新時間：2026-03-28

Dujiao-Next 支援會員等級系統，為不同消費層級的使用者提供專屬定價和折扣。會員等級可自動升級（基於充值/消費累計），也可由管理員手動指派。

---

## 1. 功能概覽

| 功能 | 說明 |
|------|------|
| 等級管理 | 建立多個等級，設定折扣率和升級門檻 |
| 等級專屬價 | 為指定商品/SKU 設定會員專享價格 |
| 自動升級 | 充值或消費累計達標後自動升級 |
| 手動指派 | 管理員可直接設定使用者等級 |
| 等級折扣 | 全域折扣率自動套用於所有商品 |

---

## 2. 等級設定

### 2.1 建立等級

後台「會員等級」頁面建立，每個等級包含：

| 欄位 | 說明 |
|------|------|
| 名稱 | 等級名稱（支援多語言） |
| 標識 (Slug) | 唯一標識，如 `default`、`silver`、`gold`、`diamond` |
| 圖示 | Emoji 或圖示 URL |
| 折扣率 | 0-100 的數值。100 = 無折扣，90 = 九折（10% off），80 = 八折（20% off） |
| 充值門檻 | 累計充值達到此金額可自動升級（0 = 不以充值為條件） |
| 消費門檻 | 累計消費達到此金額可自動升級（0 = 不以消費為條件） |
| 排序權重 | 數值越大等級越高 |
| 預設等級 | 新使用者自動指派的等級（僅一個） |
| 是否啟用 | 開關 |

### 2.2 等級範例

```
預設會員（default）  折扣率 100  排序 0   → 無折扣
白銀會員（silver）   折扣率 95   排序 10  → 充值滿 500 或消費滿 1000
黃金會員（gold）     折扣率 90   排序 20  → 充值滿 2000 或消費滿 5000
鑽石會員（diamond）  折扣率 85   排序 30  → 充值滿 5000 或消費滿 10000
```

---

## 3. 價格計算

會員價格按以下優先順序確定：

### 3.1 優先順序

1. **SKU 級專屬價**：為特定 SKU 設定的會員價（最高優先）
2. **商品級專屬價**：為商品設定的會員價
3. **全域折扣率**：按等級折扣率計算 → `原價 × 折扣率 / 100`

> 只有會員價低於原價時才會生效，不會出現會員價高於原價的情況。

### 3.2 設定專屬價

後台「會員等級價格」頁面：

- 選擇會員等級和商品
- 可為商品設定統一會員價
- 也可為商品下的具體 SKU 單獨設定會員價
- SKU 級別的設定優先於商品級別

---

## 4. 自動升級

### 4.1 升級條件

使用者達到以下**任一**條件即可升級：

- 累計充值金額 ≥ 等級的「充值門檻」（門檻 > 0 時生效）
- 累計消費金額 ≥ 等級的「消費門檻」（門檻 > 0 時生效）

### 4.2 升級時機

以下操作完成後系統自動檢查並升級：

- 錢包充值成功
- 訂單付款成功

### 4.3 升級規則

- **只升不降**：等級只會向上升級，不會因後續消費減少而降級
- **跨級升級**：如果累計金額直接達到高等級門檻，會跳過中間等級
- 按排序權重從高到低比對，取滿足條件的最高等級

---

## 5. 管理員操作

### 5.1 手動設定等級

後台「使用者管理」中可直接設定使用者的會員等級，不受門檻限制。

### 5.2 批次回填

對於系統升級前已存在的舊使用者，可使用「批次回填預設等級」功能，為尚未指派等級的使用者統一設定預設等級。

---

## 6. 注意事項

- 每個系統只能有一個預設等級
- 預設等級不可刪除
- 等級標識（Slug）建立後不可重複
- 折扣率設為 100 或以上表示該等級無折扣
- 門檻設為 0 表示該條件不參與判斷
- 會員折扣在活動價之前套用，兩者可疊加
