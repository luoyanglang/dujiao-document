# 更新日誌

## [v0.0.7-beta] - 2026-02-28

### 更新內容：
- 新增推廣返利系統：使用者可開通聯盟 ID 進行推廣，支援點擊歸因、佣金累計與提現流程。
- 擴充支付渠道能力：新增 TokenPay，並優化 Bepusdt 設定方式，支付接入更靈活。
- 優化庫存扣減與售罄狀態：庫存與下單狀態聯動更準確，降低前後台顯示不一致問題。
- 優化支付頁面與支付流程體驗：下單支付步驟更清楚，操作回饋更直觀。
- 優化初始管理員設定流程：首次部署時管理員初始化設定更穩定、相容性更好。

## [v0.0.6-beta] - 2026-02-27

### 更新內容：
- 優化錢包充值狀態體驗：長時間未支付的充值單會按規則過期，狀態顯示更清楚。
- 提升充值回調穩定性：支付結果同步更穩，降低重複回調造成的狀態異常。
- 優化訂單狀態流轉：後台訂單狀態更新更順暢，狀態一致性更好。
- 持續優化多語言與系統穩定性：細節文案與流程體驗進一步統一。

## [v0.0.5-beta] - 2026-02-25

### 更新內容：
- 新增禮品卡體系：後台支援禮品卡生成、狀態管理、篩選、批量狀態更新與匯出；前台用戶中心新增禮品卡兌換頁（含動效），支援按驗證碼場景開關校驗（圖片驗證碼/Turnstile），兌換後自動充入錢包餘額。
- 禮品卡幣種策略調整：後台生成禮品卡時幣種自動繼承全站幣種，移除管理員手動輸入幣種，避免幣種配置不一致。
- 支付回調安全熱修：回調入口僅接受已識別且通過驗簽的渠道回調（微信/支付寶/易支付/EPUSDT），阻斷偽造回調將訂單或充值單錯誤標記為已支付；同時保留 `GET /payments/callback` 以相容易支付回調，未識別請求統一回傳 `fail` 且不會更新支付狀態。

## [v0.0.4-beta] - 2026-02-25

### 更新內容：
- 新增商品分類圖標能力：後台支援上傳分類圖標，前台分類側欄與商品卡片可顯示分類圖標。
- 新增 SKU 全鏈路支援：後台商品管理、庫存顯示與前台下單流程統一到 SKU 維度。
- 優化自動交付庫存邏輯：自動發貨商品的庫存統計與顯示規則保持一致，並修復自動庫存統計入參問題。
- 增強折扣與結算防呆：強化活動配置提醒與訂單金額校驗，降低誤配置導致異常價格或 0 元單風險。
- 增強管理員退款校驗：後台退款到餘額前新增訂單支付狀態校驗，避免未支付訂單被誤退款。
- 修復支付跳轉異常：修復部分情境下支付完成後的跳轉問題。
- 修復前端舊資源請求問題：降低發佈後請求舊 chunk 導致頁面報錯的機率。
- 統一全站幣種欄位：前後台與介面幣種欄位與顯示邏輯統一。
- 新增 SEO Meta 配置能力：支援配置更完整的站點 SEO 中繼資訊。
- 體驗與穩定性優化：包含後台管理、卡密管理、購物車體驗與郵件發送鏈路等多項改進。

## [v0.0.3-beta] - 2026-02-22

### 更新內容：
- 新增用戶錢包體系：支援餘額、充值、錢包明細流水記錄（含充值、支付、退款回退等資金動向）。
- 升級訂單支付能力：支援僅餘額支付、餘額+線上支付組合支付、僅線上支付；訂單未支付成功或取消時自動退回已扣餘額。
- 後台新增錢包管理能力：支援查看用戶餘額與明細、手動增減用戶餘額、訂單退款僅退回用戶餘額。
- 新增 Telegram 登錄能力：首次 Telegram 登錄且未綁定帳號時自動建立帳號並直接登錄。
- 安全中心新增 Telegram 綁定/解除綁定：已登錄用戶可綁定或解除綁定 Telegram；未綁定真實郵箱的 Telegram 新帳號不允許解除綁定。
- 完善帳號安全策略：新增 `email_change_mode` 與 `password_change_mode` 返回字段，支援「僅綁定新郵箱」與「首次設置密碼無需舊密碼」流程。
- 優化用戶中心導覽：左側選單順序調整為「概覽 / 我的訂單 / 我的錢包 / 安全中心 / 資料設定」，並補充選單圖標。
